# Recruiter Talk Track

## 30-second summary

This GitHub project demonstrates my defense-in-depth approach to enterprise security. It reflects my background as an AVP IT Networks and Infrastructure Support Manager, where I worked across network security, firewalls, infrastructure availability, endpoint protection, vulnerability management, logging, and audit remediation. The project shows how I translate hands-on infrastructure experience into a structured cybersecurity architecture.

## 60-second summary

I created this project to show recruiters that my experience is not limited to operating devices. I understand how controls fit together: governance, identity, network segmentation, endpoint security, vulnerability management, monitoring, incident response, and recovery. The architecture assumes breach and uses layered controls to limit compromise, detect suspicious activity, and recover safely. It is mapped to NIST CSF 2.0 and CIS Controls v8 so that technical implementation is connected to recognised security outcomes.

## Interview points

- I have managed and supported enterprise network and infrastructure environments where uptime, security, and audit evidence mattered.
- I understand perimeter and internal security controls, including firewalling, VPN, segmentation, routing, logging, and endpoint protection.
- I can communicate with both technical engineers and senior stakeholders.
- I can document security architecture, operational controls, incident response, risk registers, and implementation roadmaps.
- I am comfortable moving between infrastructure engineering, network security, and cybersecurity operations.
