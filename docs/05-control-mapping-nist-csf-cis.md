# Control Mapping: NIST CSF 2.0 and CIS Controls v8

This project aligns security activities with two recognised control frameworks.

## NIST CSF 2.0 mapping

| NIST CSF function | Project implementation |
|---|---|
| Govern | Risk register, policies, change control, audit tracking, third-party reviews |
| Identify | Asset inventory, network diagrams, data classification, dependency mapping |
| Protect | Firewalls, segmentation, MFA, endpoint protection, patching, encryption |
| Detect | SIEM logging, correlation rules, endpoint alerts, vulnerability scanning |
| Respond | Incident response playbook, escalation matrix, containment procedures |
| Recover | Backup strategy, restore testing, disaster recovery runbook |

## CIS Controls v8 mapping

| CIS Control area | Project implementation |
|---|---|
| Inventory and Control of Enterprise Assets | Asset register, network discovery, ownership mapping |
| Inventory and Control of Software Assets | Approved software list, patching, endpoint inventory |
| Data Protection | Encryption, backup protection, data retention, sensitive data handling |
| Secure Configuration | Network and endpoint hardening baselines |
| Account Management | RBAC, privileged access review, leavers process |
| Access Control Management | MFA, conditional access principles, VPN access control |
| Continuous Vulnerability Management | Authenticated scans, remediation SLAs, risk acceptance |
| Audit Log Management | Central logging, SIEM, retention and review |
| Email and Web Browser Protections | Anti-phishing, attachment scanning, web filtering |
| Malware Defences | EDR/AV, alert triage, incident response |
| Data Recovery | Backup schedule, restore testing, offline/immutable copies |
| Network Infrastructure Management | Secure routing, switch hardening, configuration control |
| Network Monitoring and Defence | Firewall logs, IDS/IPS, SIEM use cases |
| Security Awareness | Phishing awareness and user reporting procedures |
| Service Provider Management | Third-party connectivity and access review |
| Incident Response Management | Playbooks, escalation, lessons learned |
| Penetration Testing | Periodic validation and remediation tracking |

## Why this matters

Recruiters often see tool names on CVs. This project shows how those tools and skills fit into a structured control model that reduces business risk.
